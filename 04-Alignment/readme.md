## Q25. RLHF 的完整三步流程是什么？

**标准回答（必须流畅讲出）：**

> "RLHF 包含三个步骤：
>
> **Step 1: SFT（Supervised Fine-Tuning）**
> 用高质量的指令-回答数据对预训练模型进行监督微调，得到一个初步对齐的策略模型 π_SFT。
>
> **Step 2: 训练奖励模型（Reward Model, RM）**
> 1. 对同一个 prompt，用 SFT 模型生成多个（通常 4-9 个）不同的回答
> 2. 人类标注员对这些回答按偏好排序
> 3. 用这些排序数据训练一个奖励模型 r(x,y)，输入 prompt+回答，输出标量奖励值
> 4. 训练目标：Bradley-Terry 模型，P(y_i > y_j) = σ(r(x,y_i) - r(x,y_j))
>
> **Step 3: PPO 强化学习优化**
> 1. 用 SFT 模型初始化策略网络 π_θ（要训练的 LLM）
> 2. 对每个 prompt，策略网络生成回答 y
> 3. 奖励模型给出奖励 r(x,y)
> 4. 加入 KL 散度惩罚：R_final = r(x,y) - β * KL(π_θ || π_ref)
> 5. 用 PPO 算法更新策略网络
>
> **为什么要加 KL 惩罚？**
> 防止策略网络为了获得高奖励而偏离原始分布太远（reward hacking），比如生成无意义但被奖励模型误判为高分的回答。"
```
PPO
1. 采样：Actor 生成一个回答 y
2. 打分：Reward Model 给 y 打最终分
3. 估分：Critic 预测每一步状态的未来得分 V
4. 算优势：A = 实际得分 - V
5. 更新：用 PPO-clip 更新 Actor + Critic

GRPO
1. 采样：同一 prompt，Actor 生成 G 个回答 {y_1,...,y_G}
2. 打分：Reward Model 给每个回答打分
3. 算优势：A_i = (R_i - 组内平均分) / 组内标准差
4. 更新：用 PPO-clip 只更新 Actor

PPO:  需要额外训练一个 Critic 来估分
GRPO: 用同一组的回答互相比较，谁比谁好就行，不需要 Critic

符号	含义	作用
π_θ	要训练的语言模型	生成回答
π_ref	SFT 模型（冻结）	防止跑偏
RM	奖励模型	给回答打分
KL 惩罚	新旧策略差异	别改太离谱
clip	限制更新幅度	训练稳定
advantage	某个 token 比平均好多少	指导优化方向

```

## Q26. PPO 和 GRPO 的核心区别是什么？（阿里三面必考）

| 维度 | PPO | GRPO |
|------|-----|------|
| **Critic/Value Model** | 需要独立的 Value Network | 不需要 Value Network |
| **优势函数估计** | GAE（需要 Critic 估计 V(s)） | 组内相对标准化 |
| **显存占用** | 高（需存 Actor/LLM + Critic/RM + RM + Reference） | 低（无需 Critic） |
| **样本效率** | 需大量经验数据 | 对同一 prompt 采样一组回答 |
| **来源** | Schulman et al., 2017 | DeepSeekMath, 2024 |

**GRPO 的核心机制（必须能手写伪代码）：**

> "GRPO（Group Relative Policy Optimization）的核心思想是：**取消 Critic 模型**，对同一个 prompt 采样一组（group）回答（如 G=8 个），用组内的相对得分来计算优势函数。
>
> **算法流程**：
> ```
> 对于每个 prompt x:
>   1. 用当前策略 π_θ 生成 G 个回答 {y_1, ..., y_G}
>   2. 对每个 y_i 计算奖励 R(y_i|x)（来自奖励模型或规则奖励）
>   3. 计算组内均值: μ_R = (1/G) * Σ R(y_i|x)
>   4. 计算组内标准差: σ_R = sqrt((1/G) * Σ (R(y_i|x) - μ_R)^2)
>   5. 标准化优势: A_i = (R(y_i|x) - μ_R) / σ_R
>   6. 用 PPO-clip 损失更新策略: L = -min(r_i * A_i, clip(r_i) * A_i)
>      其中 r_i = π_θ(y_i|x) / π_old(y_i|x)
> ```
>
> **为什么 GRPO 不需要 Value Model？**
> 传统的 PPO 需要 Critic 估计每个状态的价值 V(s) 来计算优势函数 A = Q - V。GRPO 用**组内相对排名**代替绝对价值估计——一个回答的好坏是相对于同组其他回答而言的，不需要知道它的绝对价值。
>
> **GRPO 适合什么场景？**
> - 奖励信号离散的场景（如医疗诊断正确/错误）
> - 显存受限的场景（少一个 Critic 模型）
> - 需要稳定训练的场景（Critic 训练本身也是一个挑战）"

## Q27. GRPO 为什么不需要 value model？详细推导

**深入回答：**

> "GRPO 不需要 value model 的根本原因是它用**组内样本的统计特性**来估计基线（baseline），替代了 Critic 的价值估计。
>
> **传统 PPO 的问题**：
> 优势函数 Â(s,a) = Q(s,a) - V(s)。其中 V(s) 需要 Critic 网络来近似估计。Critic 的训练本身是一个回归问题，如果估计不准，优势函数就有偏差，导致策略更新方向错误。同时 Critic 增加了约 1/3 的参数量和显存占用。
>
> **GRPO 的替代方案**：
> 对同一 prompt x 采样 G 个回答 {y_1, ..., y_G}，获得奖励 {R_1, ..., R_G}。
> 用**组内均值**作为基线：baseline = μ_R = (1/G) Σ R_i
> 优势函数简化为：A_i = R_i - μ_R（去均值）
> 进一步标准化：A_i = (R_i - μ_R) / σ_R（除以标准差）
>
> **数学合理性**：
> 这种估计方式等价于假设同一组内样本的价值相同（都是基线），差异只体现在动作选择上。虽然不是无偏估计，但在实践中足够稳定，且大幅简化了训练。
>
> **适用条件**：
> - G 不能太小（通常 ≥ 4），否则统计估计不准
> - 奖励信号有一定区分度（如果所有回答奖励都一样，A_i = 0，无法更新）"

## Q28. DPO（Direct Preference Optimization）的原理和损失函数推导

**核心洞察（一句话）：**

> "DPO 证明了**不需要显式训练奖励模型**！通过偏好数据的似然比，可以直接隐式定义奖励函数，从而简化了 RLHF 的流程。"

**完整推导（必须能手写）：**

```
Step 1: Bradley-Terry 模型定义偏好概率
P(y_w ≻ y_l | x) = σ(r(x, y_w) - r(x, y_l))
其中 y_w 是人类偏好的回答，y_l 是较差的回答，σ 是 sigmoid

Step 2: 带 KL 约束的 RL 目标（标准形式）
max_π E[r(x,y)] - β * KL(π(y|x) || π_ref(y|x))
其中 π_ref 是 SFT 模型（参考策略），β 控制偏离程度

Step 3: 闭式解（DPO 的核心发现！）
对 KL 约束优化问题，最优策略有闭式解：
π*(y|x) = (1/Z(x)) * π_ref(y|x) * exp((1/β) * r(x,y))
其中 Z(x) 是归一化常数（配分函数）

由此反解出奖励函数：
r(x,y) = β * log(π*(y|x) / π_ref(y|x)) + β * log(Z(x))

Step 4: 代入 BT 模型，消去 r，得到 DPO Loss
将 r(x,y) 的表达式代入 Bradley-Terry 模型：

P(y_w ≻ y_l) = σ(β * log(π(y_w|x)/π_ref(y_w|x)) - β * log(π(y_l|x)/π_ref(y_l|x)))
              = σ(β * log(π(y_w|x)/π(y_l|x)) - β * log(π_ref(y_w|x)/π_ref(y_l|x)))

取负对数似然作为损失函数（面试必须默写）：

L_DPO(π_θ; π_ref) = -E[(x,y_w,y_l)~D] [log σ(β * log(π_θ(y_w|x)/π_ref(y_w|x)) 
                                            - β * log(π_θ(y_l|x)/π_ref(y_l|x)))]
```

**DPO 的优缺点：**

| 优点 | 缺点 |
|------|------|
| 无需 Critic/Reward Model，训练简单 | 对长序列容易欠优化 |
| 训练稳定，没有 PPO 的超参调优问题 | 容易过拟合到偏好数据分布 |
| 实现简单（一行代码） | 偏好数据需要覆盖全面 |
| 与 SFT 使用相同的训练框架 | 长文本生成任务的优化效率低 |

## Q29. DPO 和 GRPO 在**项目指标**上有什么区别？数据上有什么区别？

**项目指标对比：**

| 维度 | DPO | GRPO |
|------|-----|------|
| **训练稳定性** | 高（没有 Critic，目标函数简单） | 中（需要调 group size 和 clip epsilon） |
| **显存占用** | 中（需存 reference model） | 低（无需 Critic） |
| **长文本生成** | 欠优化（只对数概率比） | 相对更好 |
| **奖励信号** | 偏好对（排序） | 标量奖励（可来自 RM 或规则） |
| **超参敏感度** | 低（主要调 beta） | 中（group size, clip range） |

**数据上区别：**

> "**DPO 的数据**：成对的偏好数据 (x, y_w, y_l)，即同一个 prompt 的两个回答，一个更好一个更差。数据格式是三元组。
>
> **GRPO 的数据**：需要标量奖励信号。可以是：
> 1. 奖励模型打分
> 2. 规则奖励（如代码执行结果、数学题正确性）
> 3. 人工评分
> 数据格式是 (x, y, R)，对同一个 x 采样多个 y 组成 group。
>
> **在医疗场景中的选择**：
> 如果任务有明确的正确/错误判断（如诊断是否正确），用 GRPO + 规则奖励更合适；如果任务是开放性的质量评估（如报告生成质量），用 DPO + 人类偏好排序更合适。"

## Q30. 为什么要做强化学习？为什么不能直接把想要的加到 SFT 里？

**核心回答：**

> "SFT 和 RLHF 解决的问题不同，主要有三个原因：
>
> **1. 数据覆盖度有限**
> SFT 只能学习训练数据中显式包含的知识和模式。人类偏好的维度很多（安全、有用、诚实、无害），很难通过有限的 SFT 数据覆盖所有情况。
>
> **2. SFT 无法区分质量**
> SFT 的损失函数对每个训练样本一视同仁，好的回答和差的回答（如果都在训练集中）被同等对待。模型可能同时学会正确和错误的回答模式。
>
> **3. 多维偏好的权衡**
> 人类偏好是多维度的（如创意性 vs 安全性、简洁性 vs 完整性），这些维度之间可能存在冲突。SFT 的单一损失函数无法捕捉这种复杂的权衡关系。RLHF 通过奖励模型隐式编码了这些权衡。
>
> **类比**：
> SFT 相当于给学生标准答案让他背诵；RLHF 相当于让学��在练习中得到老师的反馈，学会什么样的回答更受人类喜欢。"

## Q31. RL 训练前期学不动怎么办？

**诊断和解决方案：**

> "RL 训练前期学不动是一个常见问题，可能的原因和解决方案：
>
> **1. 奖励信号问题**
> - 奖励模型打分过于集中（所有回答得分差不多）→ 优势函数接近 0
> - 解决：检查奖励模型的区分度，必要时重新训练 RM
>
> **2. KL 惩罚过大**
> - β 值设置过大，策略不敢偏离参考策略
> - 解决：逐渐减小 β（如从 0.1 降到 0.01），或用 KL 退火
>
> **3. 学习率问题**
> - RL 的学习率通常需要比 SFT 大
> - 解决：增大学习率，或用 warmup
>
> **4. PPO clip range 问题**
> - epsilon 太小，策略更新幅度受限
> - 解决：适当增大 epsilon（如从 0.1 到 0.2）
>
> **5. 奖励缩放问题**
> - 奖励值范围不合适（太大或太小）
> - 解决：对奖励做标准化（减去均值除以标准差）
>
> **6. 冷启动问题**
> - SFT 模型质量不够，生成的回答本身就很差
> - 解决：先提升 SFT 质量，或用拒绝采样（RFT）做中间步骤"

## Q32. SFT Cold Start 是什么？为什么重要？

> "SFT Cold Start 指的是**在 RLHF 之前，先用高质量数据做 SFT 得到一个基础策略模型**。
>
> **为什么重要**：
> 1. 如果直接用预训练模型做 RL，生成的回答质量太低，奖励模型难以有效区分
> 2. SFT 让模型学会基本的指令遵循能力和对话格式
> 3. SFT 模型作为 RL 的初始化（π_ref），KL 惩罚以此为基础
>
> **在实践中的技巧**：
> - Cold Start 数据的质量比数量更重要
> - 通常需要包含拒绝回答的能力（'我无法回答这个问题'）
> - 多轮对话数据比单轮数据更有价值"

## Q33. TRPO / PPO / GRPO 的关系和演进

> "**TRPO**（Trust Region Policy Optimization）是前身，通过约束 KL 散度来保证策略更新的稳定性。但 TRPO 需要计算 Fisher 信息矩阵，计算代价高（O(n³)）。
>
> **PPO** 是 TRPO 的简化版，用 clip 机制替代 KL 约束，用一阶优化达到类似效果，更适合大规模训练。
>
> **GRPO** 进一步简化了 PPO，取消了 Critic 模型，用组内相对排名计算优势，适合 LLM 训练场景。"
