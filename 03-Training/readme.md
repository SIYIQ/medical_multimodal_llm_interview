## Q20. 预训练（Pre-training）的目标函数是什么？

**核心回答：**

> "Decoder-only 大模型的预训练目标函数是**下一个 token 预测（Next Token Prediction）**，本质上是一个分类问题——在给定前文的情况下，预测下一个位置应该是什么 token。
>
> **数学表达**：
> ```
> L = -sum(log P(x_t | x_<t; theta))
> ```
> 即对序列中每个位置 t，最大化模型预测真实 token x_t 的对数概率。
>
> **等价于交叉熵损失**：
> 将模型的输出 logits 经过 softmax 后与真实的 one-hot 标签（ground truth token）计算交叉熵。在实现上通常使用 torch.nn.CrossEntropyLoss。
>
> **注意点**：预训练时对整个序列的每个位置都计算损失（除了 padding 位置），而指令微调时通常只计算 assistant 回复部分的损失。"

## Q21. SFT（Supervised Fine-Tuning）的完整流程和注意事项

**回答框架：**

> "SFT 是在预训练模型基础上，使用高质量的（指令，回答）数据对进行监督学习，让模型学会遵循指令和对话格式。
>
> **数据构造**：
> 1. **指令多样性**：覆盖问答、摘要、代码、推理、翻译等多种任务
> 2. **格式统一**：使用 ChatML / Alpaca / ShareGPT 等格式模板
> 3. **质量控制**：过滤低质量样本，去重，平衡不同任务比例
>
> **训练细节**：
> 1. **学习率**：通常比预训练小 10-100 倍（如 1e-5 ~ 2e-5）
> 2. **损失计算**：只计算 assistant 回复部分的 loss，不计算 user 指令和 system prompt 部分
> 3. **数据配比**：不同任务类型的比例需要精心设计，防止模型偏向某类任务
> 4. **epoch**：通常 2-3 个 epoch，过多会导致过拟合
>
> **SFT 的局限性**：
> - 只能模仿训练数据中的回答模式
> - 无法区分回答质量（好坏都学）
> - 缺乏安全性约束（可能学会有害回答）
> → 因此需要 RLHF/DPO 进一步对齐"

## Q22. LoRA 微调的完整原理，公式推导，以及常见超参设置

**核心原理（重点，必须能手写公式）：**

> "LoRA（Low-Rank Adaptation）的核心假设是：微调时的权重更新 ΔW 是低秩（也就是线性无关的行/列数量较少
）的。即 W' = W_0 + ΔW = W_0 + BA
>
> 其中：
> - W_0 ∈ R^(d×k) 是预训练的冻结权重
> - B ∈ R^(d×r), A ∈ R^(r×k)，r << min(d,k) 是低秩矩阵
> - 只训练 B 和 A，W_0 保持不变
>
> **初始化技巧**：
> - A 用 kaiming_uniform_ 初始化
> - B 初始化为**全零**，这样训练开始时 ΔW = BA = 0，输出与预训练模型一致，保证训练稳定性
>
> **缩放因子**：实际实现中有 scaling = alpha / r，所以完整公式是 W' = W_0 + (B * A) * (alpha / r)
>
> **常见超参**：
> | 参数 | 典型值 | 说明 |
> |------|--------|------|
> | r | 8, 16, 32, 64 | 秩，越大表达能力越强，显存占用越多 |
> | alpha | 16, 32, 64 | 通常设为 r 或 r 的 2 倍 |
> | dropout | 0.0, 0.05, 0.1 | LoRA 层的 dropout |
> | target_modules | q_proj, v_proj | 通常只注入 attention 的 Q 和 V 投影层 |
> | lr | 1e-4 ~ 1e-3 | 比全参数微调的学习率大 |
>
> **你的项目话术**：
> '我在 Qwen-3B 魔改项目中使用 LoRA 进行微调，设置 r=16, alpha=32，target_modules=[\"q_proj\", \"v_proj\", \"k_proj\", \"o_proj\"]，lr=2e-4。选择这些模块是因为 attention 层对任务适配最敏感。通过 peft 库实现，训练时只更新了约 0.1% 的参数，但在下游任务上达到了接近全参数微调的效果。'"

## Q23. Q-LoRA 是什么？与 LoRA 的区别？

> "Q-LoRA 在 LoRA 的基础上增加了**量化**，进一步降低显存占用：
>
> **关键技术**：
> 1. **4-bit NormalFloat 量化**：将基模型权重量化为 4-bit，NF4 是一种对正态分布最优的非均匀量化格式
> 2. **双量化（Double Quantization）**：对量化常数本身也进行量化，进一步节省显存
> 3. **分页优化器（Paged Optimizers）**：当 GPU 显存不足时，自动将优化器状态分页到 CPU 内存
>
> **对比**：
> | 维度 | LoRA | Q-LoRA |
> |------|------|--------|
> | 基模型精度 | FP16/BF16 | 4-bit NF4 |
> | 65B 模型微调显存 | ~80G | ~40G |
> | 适合场景 | 有充足显存 | 单卡/少卡场景 |
> | 性能损失 | 无 | 通常 < 1% |
>
> **NF4 为什么比 INT4 好？**
> NF4（NormalFloat 4-bit）16个值 4个bit（16个整数值）非均匀量化 是专为服从正态分布的权重设计的非均匀量化格式。由于预训练模型的权重通常近似正态分布，NF4 的量化点密度在中间区域更高，对常见值量化误差更小，因此比均匀分布的 INT4 精度更高。"

## Q24. P-tuning / Prefix-tuning / Adapter 与 LoRA 的对比

| 方法 | 原理 | 训练参数 | 推理延迟 | 代表库 |
|------|------|---------|---------|--------|
| **LoRA** | 权重矩阵 低秩矩阵近似权重更新 | 中等（0.1%-1%） | 无（可合并） | peft |
| **Prefix Tuning** | 输入前加前缀 在输入前添加可学习前缀 前缀就像"任务提示" 只在输入层加| 较少 | 无 | peft |
| **P-Tuning v2** | 每层加提示 在 Transformer 每一层都加可学习提示 | 较少 | 无 | peft |
| **Adapter** | 在 FFN 后插入小型 Bottleneck 模块 | 较少 | 有增加 | adapter-transformers |
| **Prompt Tuning** | 只在输入层添加可学习 token 输入: [可学习prompt] [真实输入] | 最少 | 无 | peft |
