## Q34. 什么是 AI Agent？与大模型直接调用有什么区别？

**核心区别：**

> "大模型直接调用是**单轮响应模式**：输入 prompt → 生成回答。Agent 是**自主决策模式**：观察环境 → 思考规划 → 执行动作 → 观察反馈 → 循环。
>
> **Agent 的核心组件**：
> 1. **Perception（感知）**：接收环境输入（用户问题、工具返回结果等）
> 2. **Planning（规划）**：分解任务，制定执行计划
> 3. **Memory（记忆）**：短期记忆（对话历史）+ 长期记忆（知识库、经验）
> 4. **Tool Use（工具调用）**：调用外部工具（搜索、代码执行、API）
> 5. **Action（执行）**：输出动作（回答用户、调用工具等）
>
> **类比**：
> 大模型是'大脑'，Agent 是'大脑 + 手脚'。Agent 让大模型具备了自主解决问题的能力。"

## Q35. ReAct / Reflexion / CoT 的区别和应用场景

| 框架 | 核心思想 | 特点 | 适用场景 |
|------|---------|------|---------|
| **CoT** | 让模型'一步步想'，生成中间推理步骤 | 只推理，不行动 | 数学推理、逻辑推理 |
| **ReAct** | Reasoning + Acting 交替进行 | Thought → Action → Observation 循环 | 需要工具调用的任务 |
| **Reflexion** | 失败后自我反思，总结经验 | 有记忆和学习能力 | 复杂多步任务 |
| **ToT** | 维护多个推理路径，搜索最优 | 树状搜索 | 需要探索的复杂问题 |

**ReAct 的详细流程：**

> "ReAct 的每一步遵循以下循环：
> 1. **Thought**：模型思考当前状态和下一步计划
> 2. **Action**：模型选择一个动作（如调用工具、回答问题）
> 3. **Observation**：执行动作后获得反馈
> 4. 重复 1-3 直到任务完成
>
> **示例**（医疗场景）：
> - Thought: 用户描述胸痛症状，我需要先获取更多信息
> - Action: 询问疼痛持续时间
> - Observation: 用户回复'已经持续3天'
> - Thought: 持续3天的胸痛需要排查心脏问题，建议做心电图
> - Action: 给出就医建议"

## Q36. Agent 多轮对话怎么不忘记最开始任务？

> "这是 Agent 的**长期一致性**问题，解决方案：
>
> **1. 原始任务缓存**：
> 在 system prompt 或记忆模块中始终保留原始任务的描述，每轮都作为上下文输入。
>
> **2. 摘要机制**：
> 当对话历史太长时，定期生成摘要，保留关键信息和原始目标。
>
> **3. 目标检查点**：
> 在关键步骤设置检查点，验证当前行动是否与原始目标一致。如果不一致，触发回溯。
>
> **4. 显式目标追踪**：
> 维护一个'当前目标栈'，每次子任务完成后弹栈回到上级目标。
>
> **你的 Beyond Persona 论文话术**：
> '在我的 Beyond Persona 研究中，我设计了一个四组件的认知控制架构，其中 Boundary Reasoning 模块专门用于检测模型是否偏离了原始角色设定。这个机制可以迁移到 Agent 场景——将角色画像替换为任务目标，当检测到偏离时触发 Experiential Learning 模块进行纠正。这本质上是一种元认知监控机制。'"

## Q37. Agent 怎么做记忆（Memory）和自我改进？

**记忆系统设计：**

> "**短期记忆**：滑动窗口保存最近 N 轮对话，超过窗口的部分做摘要。
>
> **长期记忆**：
> 1. **向量数据库**：将历史对话 embedding 后存入向量数据库，通过相似度检索相关经验
> 2. **知识图谱**：结构化存储实体关系（如'症状-疾病-检查'的关联）
> 3. **经验缓存**：存储成功/失败的案例模式
>
> **自我改进机制**：
> 1. **反射（Reflection）**：任务完成后，分析成功/失败原因，生成经验教训
> 2. **经验写入**：将经验教训存入长期记忆
> 3. **未来引用**：遇到类似任务时检索相关经验，调整策略
>
> **医疗场景应用**：
> 当 Agent 成功辅助诊断一个罕见病例后，将'症状组合→诊断思路→关键检查'的经验写入知识图谱。下次遇到类似症状组合时自动检索并推荐。"

## Q38. MCP 协议是什么？为什么重要？

> "MCP（Model Context Protocol）是 Anthropic 提出的**开放协议标准**，定义了 AI Agent 与外部工具/数据源交互的标准接口。
>
> **为什么重要**：
> 1. **标准化**：之前每个 Agent 框架都有自己的工具调用格式，MCP 统一了接口
> 2. **互操作性**：遵循 MCP 的工具可以被任何支持 MCP 的 Agent 使用
> 3. **安全性**：协议内置了权限控制和数据验证
> 4. **生态**：正在形成工具生态，开发者可以发布 MCP 兼容的工具
>
> **核心概念**：
> - **Server**：提供工具/资源的端点
> - **Client**：调用工具的 Agent
> - **Capability**：描述工具的能力 schema"

## Q39. 设计一个医疗诊断 Agent（场景开放题）

**完整回答框架：**

> "我设计的医疗诊断 Agent 包含以下核心组件：
>
> **1. 感知层（Perception）**
> - 多模态输入处理：CT/X光影像通过 VLM（如 Qwen2.5-VL）编码，文本通过 LLM 编码
> - 电子病历解析：提取结构化信息（年龄、病史、检验指标）
>
> **2. 推理层（Reasoning）**
> - 采用 ReAct 范式：Thought → Action → Observation
> - 多步诊断推理：先观察影像 → 再检索知识 → 综合判断
> - 引入我在 Beyond Persona 中验证的 Contrastive Deliberation 思想：生成多个候选诊断方案，对比选择最优
>
> **3. 记忆层（Memory）**
> - 短期记忆：滑动窗口维护对话上下文
> - 长期记忆：通过 RAG 接入医学知识图谱（UMLS）和临床指南
> - 参考 HR-GNN 中的知识原型设计：用预训练编码器将指南编码为知识节点
>
> **4. 工具层（Tool Use）**
> - 影像分析工具：自动分割、特征提取
> - 知识检索工具：RAG 检索相关文献和指南
> - 计算工具：BMI 计算、风险评分
>
> **5. 安全层（Safety）**
> - 边界检测：当置信度低于阈值时，标记为'不确定'并推荐转诊
> - 幻觉检测：对照知识库验证生成内容的准确性
> - 高风险拦截：疑似恶性肿瘤等紧急情况必须建议就医"
